## Nano Banana を API 経由で利用してラフ絵から完成絵を生成する仕組み

### 実装予定なもの
ユーザーがアップロードした **ラフ絵** と、指定した **色情報**、任意の **アスペクト比**・**解像度** を API に送信することで、Nano Banana による **完成イラスト生成** を行う。

### 入力項目
- **画像（ラフ絵）**  
  ユーザーが描いた下書き画像をアップロードする。
- **色指定**  
  ラフ絵に着色してほしい色（帽子は赤、服は黒 など）を指定する。
- **アスペクト比（任意）**  
  例: `1:1`, `16:9`, `4:5` など。APIの仕様で指定されているものの中から洗濯するようにする
- **解像度（任意）**  
  例: APIの仕様で指定されているものの中から選択するようにする

### 処理の流れ
1. ユーザーがラフ絵と指定パラメータをフォームまたはアプリケーションに入力する。  
2. アプリケーションはそれらの情報を Nano Banana API にリクエストとして送信する。  
3. Nano Banana がラフ絵を解析し、色指定・アスペクト比・解像度に従って完成イラストを生成する。  
4. 完成イラストをレスポンスとして受け取り、ユーザーに表示・ダウンロードさせる。

### 届いている要望
- キャラクターの設定（瞳の形状・カラーコードなど）記憶させ、簡易呼び出し名で出力させる（Ω、βみたいな日本語で使わない言葉がキャラ指定コードになる、とか...）
- 背景とキャラと吹き出しを分けて出力（手直しが格段にしやすくなる）
- Aの線画をBの画像の色や雰囲気で完成させる。（作者のタッチの再現）
- 完成絵の微修正（指や表現、背景のみ、など）
- 変化の程度。（ラフ絵の雰囲気のまま加筆・ラフ絵を元に大幅に改変等）
- コピーした画像をそのままペーストして貼れると楽
